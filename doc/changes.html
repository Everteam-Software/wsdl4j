  • Version 1.5.2
    Classes Affected Changes
    com.ibm.wsdl.xml.WSDLReaderImpl v1.41 - Added verbose trace to parseImport to differentiate schemas imported by <wsdl:import> from those imported by <xsd:import>

  • Version 1.5.1
    Classes Affected Changes
    com.ibm.wsdl.xml.WSDLReaderImpl In parseOperation, fixed a logic error that was incorrectly causing duplicate operation errors.

    In parseExtensibilityElements, added a check that the extensibility element is not in the WSDL namespace.

    Set base URI on InputSource so relative URIs get resolved when parsing the Document. Made this change to readWSDL, parseImport and parseSchema methods. Also, corrected related error messages to use full URI where possible, not just the relative URI.

    Fixed NPE when WSDLLocator used in parseImport by ensuring base URI is set on InputSource - thanks to Pete Hendry and David Jencks for their suggestions.

    In method readWSDL(WSDLLocator), set the base URI of the InputSource so that subsequent relative URIs will be resolved correctly. Also changed method readWSDL(String,InputSource), to use this URI if it is present.

    com.ibm.wsdl.util.xml.DOMUtils Corrected error checking to ignore namespace declarations when checking for unexpected extensibility attributes.
